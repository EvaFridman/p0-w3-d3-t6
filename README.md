# p0-w3-d3-t6 · Нормализация телефонов

> Тип: регулярки · Репозиторий: один отдельный

> Подключить файл, отсмотреть в браузере.

## Задача

Телефоны в базу попадают в каком угодно виде: `8(999)123-45-67`, `+7 999 1234567`, `89991234567`. Чтобы их можно было сравнивать и хранить, всё приводят к одному формату. Это делается регуляркой с группами захвата и `replace`.

```
телефоны в разном виде → единый формат
  '8(999)1234567' ┐
  '+7 999 123 45 67' ├→ '+7 999 123-45-67'
  '89991234567'   ┘
replace(re, fn) + группы захвата
```

## Что нужно сделать

В `phones.js` взять готовый массив телефонов (ниже) и привести каждый к единому виду `+7 999 123-45-67`.

Ключевой приём: вторым аргументом `replace` можно передать не строку, а **функцию** — она получает совпадение (и группы захвата, если они есть) и возвращает строку-замену. Пригодится, чтобы собрать номер в нужном виде из вытащенных цифр. Мини-иллюстрация на постороннем примере:

```js
"a1b2".replace(/\d/g, (d) => `[${d}]`); // → "a[1]b[2]"
```

Регулярку для телефона и сборку формата пишешь сама.

Входные данные:

```js
const phones = [
  "8(999)123-45-67",
  "+7 999 1234567",
  "89991234567",
  "7 (999) 123 45 67",
  "12345",              // заведомо не телефон
];
```

Ожидаемый результат:

```
"8(999)123-45-67"   → "+7 999 123-45-67"
"+7 999 1234567"    → "+7 999 123-45-67"
"89991234567"       → "+7 999 123-45-67"
"7 (999) 123 45 67" → "+7 999 123-45-67"
"12345"             → "не похоже на телефон"
```

## Критерии приёмки

* Телефоны в разных исходных форматах приводятся к одному; используется `replace` с группами захвата.
* Корректно обрабатывается ведущая `8` → `+7` и разные разделители/скобки.
* Заведомо «не телефон» отсекается или помечается, а не превращается в мусор.
